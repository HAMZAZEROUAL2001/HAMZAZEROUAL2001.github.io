Project Overview

This educational project demonstrates how ransomware works at a fundamental level. It's designed to help security researchers, students, and professionals understand the mechanics behind ransomware attacks to better defend against them.

The project implements basic file encryption, ransom note generation, and key management, showing how attackers can encrypt user data and demand payment for decryption. This knowledge is crucial for developing effective detection and prevention strategies.

How Ransomware Works

1. Initialization Phase

The program generates a cryptographically secure encryption key using a random number generator. This key is used for all file encryption operations.

2. File Discovery

The ransomware recursively scans directories to identify target files based on extensions (.docx, .pdf, .jpg, .png, .db, etc.). It builds a list of files to encrypt.

3. Encryption Process

Each target file is encrypted using AES-256 in CBC mode. The original file is overwritten with encrypted data, and the file extension is changed (e.g., .docx.encrypted).

4. Ransom Note Deployment

A ransom note text file is created in each affected directory, informing the "victim" of the encryption and providing instructions (in a real attack, this would include payment details).

Code Example - File Encryption

// Simplified encryption function
void encryptFile(const char* filename, const unsigned char* key) {
    FILE *inFile = fopen(filename, "rb");
    FILE *outFile = fopen(tempfile, "wb");
    
    // Initialize AES context
    AES_KEY aesKey;
    AES_set_encrypt_key(key, 256, &aesKey);
    
    unsigned char buffer[AES_BLOCK_SIZE];
    unsigned char encrypted[AES_BLOCK_SIZE];
    size_t bytesRead;
    
    // Encrypt file block by block
    while ((bytesRead = fread(buffer, 1, AES_BLOCK_SIZE, inFile)) > 0) {
        AES_encrypt(buffer, encrypted, &aesKey);
        fwrite(encrypted, 1, AES_BLOCK_SIZE, outFile);
    }
    
    fclose(inFile);
    fclose(outFile);
    
    // Replace original with encrypted
    remove(filename);
    rename(tempfile, strcat(filename, ".encrypted"));
}

Typical Ransomware Attack Chain

  • Initial Access: Phishing email, malicious download, or exploit
  • Execution: Malware runs with user privileges
  • Persistence: Creates registry keys or scheduled tasks
  • Defense Evasion: Disables antivirus, deletes shadow copies
  • Discovery: Scans for valuable files and network shares
  • Encryption: Encrypts files using strong cryptography
  • Ransom Demand: Displays note and demands payment

Defense & Prevention Strategies

Preventive Measures

  • Regular Backups: Maintain offline, encrypted backups (3-2-1 rule)
  • Email Filtering: Block suspicious attachments and links
  • User Training: Educate users about phishing and social engineering
  • Patch Management: Keep systems and software up to date
  • Least Privilege: Limit user permissions and access
  • Network Segmentation: Isolate critical systems

Detection Techniques

  • Monitor for suspicious file modification patterns
  • Detect rapid, sequential file access across directories
  • Identify processes performing encryption operations
  • Monitor for volume shadow copy deletion
  • Analyze network traffic for C2 communications
  • Use behavioral analysis and machine learning

Response Actions

  • Immediately disconnect affected systems from network
  • Identify and isolate patient zero
  • Preserve forensic evidence (memory dumps, logs)
  • Contact law enforcement and cybersecurity experts
  • DO NOT pay ransom (no guarantee of decryption)
  • Restore from backups after thorough system cleaning

Technical Analysis

Encryption Analysis

This implementation uses AES-256, which is a symmetric encryption algorithm considered cryptographically secure. Without the encryption key, decryption is computationally infeasible (would take billions of years with current technology).

Key Management

In real ransomware, the symmetric key is typically encrypted with the attacker's public RSA key before being sent to a command-and-control server. This prevents reverse engineering of the key from the malware binary.

File System Impact

The ransomware overwrites original files, making recovery without the key nearly impossible. Advanced variants also delete shadow copies and disable system restore to prevent recovery.
